# jp-med-dict(仮称)

日本語の **医療辞書(薬剤成分・商品名・病名)** を公開データから自動構築し、SQLite 単一ファイルとして OSS 配布する独立プロジェクト。DICTATION(医療向け音声入力)や、その他の日本語医療 NLP アプリから利用されることを想定する。

## 目的

- **無料でアクセス可能な公的/オープンデータ**を統合し、「薬の成分・商品名」「病名(表記ゆれ含む)」を中心とした辞書を構築する。
- GitHub Actions で **月次(薬価改定時は臨時)自動更新**し、バージョンタグ付き SQLite を Releases に配布する。
- アプリ側は manifest を見て差分 DL、ローカル SQLite をアトミック置換する(ゼロダウンタイム)。

## スコープ

- 対象: **薬剤成分 / 商品名 / 病名 / 主要手技(K コード)**
- 非対象: 臓器・解剖用語・生理学用語など(成長が遅い・既存辞書で十分)

## 技術方針(暫定)

- **ETL 言語**: Python(pandas・lxml 等、スクレイピング/XML処理ライブラリが豊富)。Swift 縛りは DICTATION 側の制約であり、本プロジェクトには適用しない。
- **成果物**: SQLite 単一ファイル(圧縮: zstd)。スキーマは `schema.sql` に版管理。
- **配布**: GitHub Releases に `.sqlite.zst` + `manifest.json`(version, sha256, row counts)。
- **自動更新**: `.github/workflows/monthly-update.yml` が cron で各ソースを取得 → diff 検出 → ビルド → テスト → Release 発行。
- **消費側 API**: Swift / Python / CLI 向けの薄いクライアント(Phase 2)。

## データソース(現時点の候補)

| 種別 | ソース | ライセンス | 備考 |
|------|--------|-----------|------|
| 薬(添付文書) | PMDA 添付文書 XML | 公的公開 | 成分・商品名・規格 |
| 薬(薬価) | 厚労省 薬価基準収載品目 | 公的公開 | YJ コード・月次更新 |
| 薬(一般名) | 厚労省 一般名処方マスタ | 公的公開 | 一般名処方対応 |
| 病名 | 万病辞書 (MANBYO) | **CC BY-SA 4.0** | 表記ゆれ 12 万語超。**派生物コピーレフト伝播** |
| 病名 | DPC 傷病名マスター | 公的公開 | ICD-10 紐付き |
| 病名 | ICD-10 日本語版 | 公的公開 | |
| 手技 | 診療報酬 K コード | 公的公開 | |

### 採用しない(ライセンス・再配布理由)

- MEDIS 標準病名マスター(再配布条件が厳しいためコアから外す。将来プラグイン方式で各自追加する構成なら検討)
- KEGG DRUG(商用制限の恐れ)
- MedDRA / JAPIC(有償)
- UMLS / RxNorm / SNOMED CT(英語 / 要ライセンス)

## ライセンス方針

- **データ**: 万病辞書取り込みにより **CC BY-SA 4.0**(コピーレフト伝播)。
- **コード**: MIT(辞書データと分離可能)。
- DICTATION 本体(MIT 想定)と混ぜず**独立リポジトリ**として運用することで、ライセンス衛生を保つ。

## 運用方針

- 最小から始める。初期は **薬剤成分 + 商品名 + 万病辞書 + DPC 傷病名** の 4 点のみで MVP。
- MEDIS / ICD-10 / K コードは MVP 後に追加。
- **全自動化を目指す**: 月次 cron → diff → ビルド → テスト → Release。人手介入はライセンス変更やスキーマ破壊的変更のみ。
- バッチ更新はフル置換で十分(SQLite 数十 MB 想定)。将来肥大したら差分マイグレーションに切替。
